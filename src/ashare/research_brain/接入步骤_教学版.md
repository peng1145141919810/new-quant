# V5.1 接入步骤（教学版）

## 一、先理解这一版到底解决了什么

你前面碰到的最大问题，不是“模型太少”，而是：

1. 有些模型根本不适合全量大样本。
2. GPU 没被正式纳入模型家族。
3. 系统不区分“高产出研究”和“高成本垃圾路线”。

V5.1 就是在这三件事上动刀。

---

## 二、第一步：改配置

打开：

`configs/hub_config.v5_1.local.json`

### 最关键的 4 个路径

- `project_root`
- `train_table_dir`
- `hub_output_root`
- `execution.python_executable`

你的 Python 解释器已经固定：

```json
"C:/Users/Administrator/PyCharmMiscProject/.venv/Scripts/python.exe"
```

---

## 三、第二步：确认 GPU 档案

V5.1 已经按你的机器预置了一份：

```json
"gpu_profile": {
  "enabled": true,
  "vendor": "nvidia",
  "name": "RTX 4000 Ada",
  "vram_gb": 20,
  "lightgbm_device_type": "gpu",
  "xgboost_device": "cuda",
  "gpu_platform_id": 0,
  "gpu_device_id": 0,
  "gpu_use_dp": false
}
```

这意味着：

- LightGBM GPU 路线会优先尝试显卡。
- XGBoost GPU 路线会优先尝试 CUDA。
- 如果 GPU 路线失败，允许回退到 CPU 备胎，但会扣分。

---

## 四、第三步：理解资源护栏

这一版你最该记住的不是“模型名字”，而是“模型预算”。

例如：

- `lightgbm_gpu`：允许全量。
- `xgboost_gpu`：允许全量。
- `hist_gbdt`：允许中大样本，但默认不建议千万级硬怼。
- `extra_trees`：只允许中样本。
- `random_forest`：只允许抽样训练。
- `generated_family`：有限流。

所以以后再看到某条路线慢，不要先问“是不是死了”，而要先看：

- 它是不是被预算护栏抽样了
- 它是不是被主动跳过了
- 它是不是 GPU 失败后回退到了 CPU

这些都会写进注册表。

## 五、第四步：切运行模式

打开：

`hub/local_settings.py`

### 推荐顺序

#### 1）校验环境
```python
MODE = "validate_only"
DRY_RUN = True
MAX_CYCLES = 1
```

跑完看：

`validation.json`

重点看：
- 训练表能不能读
- `nvidia-smi` 是否成功
- `lightgbm` / `xgboost` 是否能 import

#### 2）只看计划
```python
MODE = "plan"
DRY_RUN = True
MAX_CYCLES = 1
```

跑完看：

- `cycle_plan.json`
- `cycle_manifest.csv`

#### 3）跑一轮真实 batch
```python
MODE = "batch"
DRY_RUN = False
MAX_CYCLES = 1
```

#### 4）再进持续闭环
```python
MODE = "adaptive_research_brain"
DRY_RUN = False
MAX_CYCLES = None
SLEEP_SECONDS = 0
```

## 七、跑起来后先看什么

### 1）实时进度
`live_progress.json`

看它是不是在更新。

### 2）总控状态
`controller_state.json`

看它跑到第几轮、最近预算分配到哪里。

### 3）总账本
`experiment_registry.csv`

这个表最重要。

新字段里你要重点看：
- `effective_model_family`
- `budget_action`
- `estimated_cost_units`
- `gpu_used`
- `elapsed_seconds`

### 4）策略家族
`strategy_family_state.csv`

看谁是 champion，谁是 challenger。

---

## 八、你以后怎么看“GPU 到底有没有用上”

看三层：

### 第一层：控制台
候选实验启动时会显示模型家族。

### 第二层：run_summary.json
看：
- `effective_model_family`
- `gpu_used`
- `budget_action`

### 第三层：experiment_registry.csv
看这一列：
- `gpu_used`

如果它是 `True`，说明这次实验真的走了 GPU 路线。

---

## 九、下一步不是“无脑一直跑”

你后面最正确的节奏是：

1. 先让 V5.1 稳定连续跑几轮。
2. 观察哪些路线总是：
   - 高成本
   - 低得分
   - 高回撤
3. 再缩小差路线预算，放大优路线预算。
4. 从策略池里筛出 champion/challenger。
5. 最后再讨论模拟盘和实盘。

也就是说，研究中枢负责不断产出候选认知，
你负责审阅这些认知，不是把系统丢那儿不管。
