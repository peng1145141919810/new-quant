# V6 双脑 LLM 分工

## 一、原则

你希望：

- DeepSeek 干脏活
- OpenAI 的 GPT-5.4 干真正值钱的事

V6 直接按这个架构设计，不走“一个模型包打天下”的低级路线。

## 二、DeepSeek 的职责（低成本执行脑）

负责：
- 原始文本清洗
- 事件去重
- 标题归一化
- 实体识别初稿
- 事件类型分类
- 利空 / 利好 / 中性粗判
- 影响范围初判
- 影响时长初判
- 结构化事实抽取
- 批量事件摘要
- 数据补数候选任务草案
- 为 GPT-5.4 生成 compact context

不负责：
- 最终研究结论
- 预算最终分配
- 是否暂停一条策略线
- 是否新增正式特征
- 直接写进实盘动作

## 三、GPT-5.4 的职责（高价值研究脑）

负责：
- 从事件流里挑出真正值得研究的事件
- 判断影响的是：
  - 特征
  - 标签
  - 模型
  - 组合
  - 风控
  - 数据补全
- 决定本轮研究预算重心
- 决定哪些实验先做、哪些不做
- 决定新增哪些候选特征
- 决定哪些 branch 需要暂停或冻结
- 决定何时把某类事件纳入长期因子体系

GPT-5.4 的输入必须是“证据包”，不允许裸聊。

## 四、建议的调用顺序

1. DeepSeek：事件抽取
2. DeepSeek：批量压缩
3. DeepSeek：数据补数候选
4. GPT-5.4：研究计划
5. GPT-5.4：动作清单

## 五、建议的成本控制规则

### DeepSeek 高频调用
- 每天可多次
- 以结构化任务为主
- 可以并行切批

### GPT-5.4 低频高价值调用
- 每日 1~2 次主研究计划
- 每轮 cycle 前 1 次计划更新
- 重大事件发生后 1 次专项审阅
- 不做大批量逐条事件处理

## 六、双脑冲突时的处理

- DeepSeek 抽取低置信 -> 送 GPT-5.4 审阅
- DeepSeek 和规则引擎冲突 -> 先过规则校验
- DeepSeek 与 GPT-5.4 冲突 -> 两份意见都落盘，最终以 GPT-5.4 的 `approved_action` 为准
