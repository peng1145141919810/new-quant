# GPU 研究脑对接说明

## 一、原则

第一阶段不直接重写 `cli.py` 的主循环，而是在其前面插一层 `orchestrator.py`（研究计划编排器）。

```text
orchestrator（研究计划编排器）
    -> 事件接入
    -> 事件抽取
    -> 数据缺口引擎
    -> 研究证据包
    -> GPT-5.4 研究计划
    -> 生成 override 文件
    -> research_bridge 覆盖 route / candidate
    -> 调用 GPU 研究脑的 plan / batch / adaptive cycle
```

## 二、GPU 研究脑需要新增消费的文件

- `bridge/llm_route_override.json`
- `bridge/candidate_override.json`
- `bridge/enriched_context.json`

## 三、最小侵入式改法

- `candidate_factory.py` 支持读取外部 candidate override
- `research_routes.py` 支持读取 route budget override
- `cli.py` 在 `_prepare_cycle` 前读入 `enriched_context.json`

## 四、推荐的初始运行方式

1. 只跑研究计划前置，不触发训练
2. 研究计划 + GPU 研究脑 plan
3. 研究计划 + GPU 研究脑 batch
4. 研究计划 + GPU 研究脑 adaptive_research_brain
