# V5.1 对接说明

## 一、原则

V6 第一阶段不直接重写 `cli_v5.py` 的主循环，而是在其前面插一层 `orchestrator_v6.py`。

```text
orchestrator_v6
    -> 事件接入
    -> 事件抽取
    -> 数据缺口引擎
    -> 研究证据包
    -> GPT-5.4 研究计划
    -> 生成 override 文件
    -> 调用 V5.1 的 plan / batch / adaptive cycle
```

## 二、V5.1 需要新增消费的文件

- `bridge/llm_route_override.json`
- `bridge/candidate_override.json`
- `bridge/enriched_context.json`

## 三、V5.1 最小侵入式改法

- `candidate_factory.py` 支持读取外部 candidate override
- `research_routes.py` 支持读取 route budget override
- `cli_v5.py` 在 `_prepare_cycle` 前读入 `enriched_context.json`

## 四、推荐的初始运行方式

1. 只跑 V6 前置，不触发训练  
2. V6 + V5.1 plan  
3. V6 + V5.1 batch  
4. V6 + V5.1 adaptive_research_brain
