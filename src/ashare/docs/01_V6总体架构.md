# V6 总体架构

## 总体定位

V6 是 **研究前置增强层 + 研究中枢升级层**。  
它不是替代 V5.1，而是先挂在 V5.1 前面，等稳定后再把关键逻辑并回主控。

## 分层架构

### A. 原始事件层 Raw Event Lake
存放未经判断的原始材料：

- 公告正文 / 标题
- 新闻正文 / 标题
- 政策新闻
- 行业快讯
- 采集时间、发布时间、来源、URL、实体线索

### B. 标准事件层 Event Store
把文本转为结构化事件：

- 主体公司
- 证券代码
- 事件类型
- 方向
- 影响范围
- 影响时长
- 置信度
- 新颖度
- 重要度
- 结构化事实

### C. 数据健康层 Data Inventory + Gap Engine
维护：

- 当前有哪些表
- 每张表最后更新时间
- 每张表缺失率
- 是否陈旧
- 是否被近期事件触发重算
- 是否存在新增特征机会

### D. 证据包层 Research Context Pack
把研究前需要的一切证据打包：

- 最近事件摘要
- 高优先事件样本
- 市场状态快照
- 最新数据缺口报告
- 最近实验结果摘要
- champion / challenger 状态
- 当前研究预算与研究空间
- 当前暴露与回撤摘要

### E. 研究规划层 Research Brief Engine
由 GPT-5.4 读取证据包，输出：

- 研究主命题
- 本轮优先事件
- 需要补的表
- 需要重算的因子
- 建议新增的因子
- 标签/模型/组合/风控改动建议
- 停止条件与禁做项
- 候选实验列表

### F. 对接层 V5 Bridge
把 `research_actions` 转为 V5.1 可消费的输入：

- route budget override
- feature/model/training/risk candidate override
- 数据刷新任务队列
- cycle-level rationale

## 主流程

```text
基础数据更新 / 事件抓取
    -> 原始事件落湖
    -> DeepSeek 抽取结构化事件
    -> 数据缺口引擎扫描
    -> 生成证据包
    -> GPT-5.4 生成研究计划
    -> V5 Bridge 覆盖 route / candidate
    -> 调用 V5.1 训练、预测、组合、回测
    -> 结果回流 registry
    -> 下一轮继续
```

## V6 的每日循环

### 早盘前
- 刷新交易日历、股票状态、基础日线
- 拉取前一日收盘后至今晨的公告与新闻
- 生成结构化事件表
- 生成当日数据缺口报告

### 收盘后
- 补齐当日日线与状态
- 更新事件表
- 重算被触发的特征
- 生成研究证据包
- 让 GPT-5.4 生成下一轮研究计划
- 调用 V5.1 跑 batch / adaptive cycle

## 第一版必须回答的问题

1. 今天发生了哪些高价值事件？
2. 这些事件影响的是哪些股票 / 行业 / 标签周期？
3. 哪些已有数据表已经不够用了？
4. 哪些特征应该重算？
5. 哪些新特征值得进入下轮候选实验？
6. 下一轮研究预算应压到哪些方向？
