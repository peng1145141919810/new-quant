# V6 执行摘要

## 一句话定义

V6 不是继续把 V5.1 做成更花哨的参数迭代器，而是把它升级成 **事件驱动的数据—研究联合中枢**。

## 核心升级

V5.1 主要做这件事：

> 基于已有训练表，诊断研究状态，分配候选预算，生成实验并跑回测。

V6 要把前面再加四层：

1. **事件接入层**  
   每天接入公告、新闻、政策、行业事件和基础状态表。

2. **事件抽取层**  
   把原始文本变成结构化事件，不再让文本直接污染研究链。

3. **数据缺口引擎**  
   自动判断哪些基础表缺失、哪些因子应重算、哪些新特征值得新增。

4. **研究规划层**  
   让 GPT-5.4 基于“证据包”生成下一轮真正的研究计划，而不是模板建议。

## 双脑策略

### DeepSeek
负责：
- 文本清洗
- 事件初筛
- 实体识别初稿
- 事件类型分类
- 利空利好粗判
- 结构化字段抽取
- 低成本批量摘要
- 数据补数候选任务草案

### GPT-5.4
负责：
- 哪些事件值得进研究队列
- 这些事件影响的是特征、标签、模型、组合还是风控
- 哪些方向要加预算，哪些方向要暂停
- 哪些数据要补，哪些特征要新增
- 输出下一轮实验清单与停止条件

## 第一阶段成果定义

V6 第一阶段不要求实盘接管，但必须达到：

- 每日自动落地原始事件表
- 每日生成结构化事件表
- 每日输出数据缺口报告
- 每轮研究前生成 `research_context_pack.json`
- 每轮研究前生成 `research_brief.json`
- 能把 `research_actions` 转成 V5.1 的 route budget / candidate override

## 这次设计刻意不保守

本蓝图没有走“小修小补”路线，而是直接假设：

- 你允许引入新的目录层级
- 你允许新增事件湖、数据健康层、研究计划层
- 你允许 LLM 真正覆盖研究计划，不只是做注释
- 你允许先做重架构，再逐步调试实现
