# V6 风险边界与治理

## 一、最危险的三件事

1. 事件流一接入就开始让 LLM 直接选股  
2. 数据补全没有清单制度  
3. LLM 不看证据包直接给建议  

## 二、V6 的硬性护栏

### 护栏 A：事件必须先结构化
原始文本不允许直接进入研究计划层。

### 护栏 B：所有研究动作必须落盘
任何新增因子、补数建议、暂停建议，都要落成 JSON 文件。

### 护栏 C：高价值判断与低价值执行分层
DeepSeek 不能拥有最终批准权。

### 护栏 D：研究计划必须有停止条件
每一轮都要写：
- 什么时候停止这条线
- 什么结果算失败
- 触发哪些信号就冻结 branch

## 三、建议的治理文件

- `banlist.json`
- `manual_review_queue.json`
- `research_decision_log.jsonl`

## 四、为什么现在先不碰实盘

因为你当前阶段最缺的不是“自动下单”，而是：

- 事件到特征的映射体系
- 研究计划的真正智能化
- 数据补全制度
- 每轮实验的因果理由

没有这些，直接接券商就是自动把混乱放大。
