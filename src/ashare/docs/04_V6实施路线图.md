# V6 实施路线图

## 总策略

不直接把 V5.1 主控砸掉重写。  
先在前面挂四个新模块，跑通旁路，再逐步并回主干。

## Phase 0：目录与制度先立住
目标：先把 V6 的落盘结构定死。

交付物：
- `raw_events/`
- `event_store/`
- `inventory/`
- `research/context_pack/`
- `research/briefs/`
- `bridge/`

## Phase 1：事件接入层
目标：每天自动拉事件与基础表。

交付物：
- 原始公告抓取器
- 原始新闻抓取器
- Tushare 基础表更新器
- 事件原始层样例文件

## Phase 2：结构化事件层
目标：让 DeepSeek 把文本变成正式事件表。

交付物：
- `event_extract.py`
- `event_store.parquet`
- 去重逻辑
- 枚举校验逻辑
- 低置信事件回审逻辑

## Phase 3：数据缺口引擎
目标：系统第一次学会“我还缺什么”。

交付物：
- `data_inventory.json`
- `data_gap_report.json`
- `refresh_tasks.json`
- `feature_recompute_tasks.json`

## Phase 4：研究证据包
目标：别再让 LLM 裸聊。

交付物：
- `research_context_pack.json`
- 市场状态快照
- 事件优先级摘要
- 最近实验摘要
- 当前 champion / challenger 摘要

## Phase 5：GPT-5.4 研究计划
目标：真正让 LLM 参与研究。

交付物：
- `research_brief.json`
- `research_actions.json`
- route budget override
- candidate override

## Phase 6：并回 V5.1
目标：让 V5.1 真正吃到 V6 的成果。

交付物：
- `v5_bridge.py`
- cycle 级 override 文件
- route 与 candidate 覆盖逻辑

## Phase 7：影子组合（不碰实盘）
目标：每天出建议组合与调仓理由，但不下单。

交付物：
- `shadow_portfolio.json`
- `shadow_changes.json`
- `risk_watchlist.json`

## 不建议现在做的事

- 不建议现在就直接接券商下单
- 不建议现在就训通用本地小大模型
- 不建议现在把所有数据源一次性全接上
- 不建议现在把 DeepSeek 和 GPT-5.4 的边界做模糊
